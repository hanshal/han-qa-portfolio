## 🚀 Agentic AI in QA — What I've Built

### Blast Radius Analysis Pipeline (Ringier Jobs Platform)

Three Claude Code skills chained into a full QA prep workflow for a 5-domain jobs platform:

```
Developer raises PR
  → /blast_radius     → Structured PDF report (domain configs, feature flags, view overrides)
  → /pre_qa_review    → Jira-ready QA comment with severity levels (blocker / confirm / advisory)
  → /qa_test_paths    → Copy-pastable test cases ready for QA
```

**Why it matters**: Jobs serves 5 live domains across Africa and Romania. Each has its own config overrides, feature flags, and scheduling rules. Before this pipeline, cross-domain impact wasn't always visible until QA or prod. Now it's surfaced automatically on every PR.

Skills live in `.claude/commands/` and `.claude/skills/` — transparent, version-controlled, and iterable.

---

### Cypress Test Suite Optimisation Skill

A custom Claude Code skill tailored to the team's Cypress setup that:
- Analyses and refactors test suite files and custom commands
- Eliminates redundancy and strengthens assertions
- Enforces project-specific rules (Alpine.js reinitialisation, existing selectors, inline comments)
- Applies Cypress best practices at scale across the full test suite

---

### Agentic Self-Healing Pipeline *(in progress)*

End-to-end pipeline designed to close the loop between test failure and fix:

1. **Playwright MCP** inspects live element locators in real-time and flags stale/flaky ones
2. **Reruns** refactored and related test cases to catch regressions early
3. **Self-heals** failing tests — investigates root cause, applies fix, reruns to confirm
4. **Auto-posts** a structured change summary to the linked Jira ticket
